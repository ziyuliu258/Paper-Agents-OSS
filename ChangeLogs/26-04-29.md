# 2026-04-29 Memory 可信认知视图增强

## 背景与目标

本次更新依据 `specs/memory.md` 中的 memory 优先级方案，目标是在不扩大 selector 事实源、不新增独立任务系统的前提下，增强 Memory V3 的可信写回、生命周期治理和只读派生视图能力。

重点目标：

- 在长期记忆写回前拦截无证据或已被 report audit 移除的 claims。
- 为 claims 增加可解释的 lifecycle 状态，帮助区分 emerging、supported、contested 和 needs_review。
- 新增基于数据库的 Memory Health、Field Map、Evidence Matrix 派生视图。
- 在前端 Profile Detail、Memory Workspace 和 Living Survey 中暴露这些治理与溯源信息。

## 影响范围

- 后端记忆数据层：`utils/memory.py`
- Claim relation 派生层：`utils/memory_claim_relations.py`
- Interpreter 写回链路：`modules/paper_interpreter/agent.py`

## 核心改动

### 1. 写回前可信校验

- 新增 `validate_memory_extraction_for_writeback()`，在 `MemoryManager.write_memories()` 前过滤不应进入长期记忆的内容。
- 过滤无 evidence 支撑的 claims。
- 根据 `report_audit` 的 removed claim 信息过滤被审查删除的 claims。
- 同步清理 synthesis 中指向已丢弃 claims 的链接，避免留下悬空引用。
- 在 WorkingMemory 中记录 `memory_writeback_validation`、保留 claim 数和丢弃 claim 数，方便报告面板或调试追溯。

### 2. Claim lifecycle 与稳定性增强

- `memory_claims` 增加 lifecycle 相关字段，并通过 lazy migration 保持旧库兼容。
- `build_claim_relations()` 在稳定性统计中输出 lifecycle 状态和原因：
  - `emerging`：已有基础证据但尚未形成多重支持。
  - `supported`：有多条 evidence 或 relation 支持。
  - `contested`：存在 contradiction、pending review 或 conflicted 状态。
  - `needs_review`：缺少 evidence，需要人工或后续论文补强。
- relation rebuild 会同步写回 `stability_score`、`last_supported_at`、`last_challenged_at` 和 lifecycle 字段。

### 3. 新增只读派生视图

- 新增 `memory_health_snapshot`：统计 unsupported、thin evidence、contested、pending review、deprecated、scope incomplete、orphan evidence 和 stale artifacts 等治理问题。
- 新增 `field_map_snapshot`：从 themes、claims、claim relations 和 review 状态构造领域簇、簇间关系和阅读入口。
- 新增 `evidence_matrix_snapshot`：按 task / dataset / metric 聚合 evidence，显式标记 method、value、setting、scope_note 等不完整字段。
- 这些视图均保存为 `memory_derived_artifacts`，不是新的可编辑事实源。

### 4. API 与前端接线

- `GET /api/profiles/{profile_id}/workspace/health`
- `GET /api/profiles/{profile_id}/workspace/field-map`
- `GET /api/profiles/{profile_id}/workspace/evidence-matrix`
- `ProfileDetailResponse` 增加 health 和 field map preview。
- 手动 evidence 创建/更新支持 `anchor_kind`、上下文和完整 `structured_signal` 往返。
- Memory Workspace 新增 `健康度`、`领域地图`、`证据矩阵` 三个只读 tab。
- Memory Workspace 的 evidence 编辑器补齐 task、method、dataset、metric、value、baseline、comparator、setting、limitation、scope note 输入。
- Profile Detail 新增 Memory Health 与 Field Map Preview 卡片。
- Living Survey 新增目录区，并在内容块中用 `<details>` 展示 claim / synthesis / review / paper 来源追溯。

## 数据结构 / API / 页面变化

- 数据库 lazy migration：`memory_claims` 新增 lifecycle 相关字段，旧库打开时自动补齐。
- API 新增三个 workspace 派生视图接口，并扩展 workspace snapshot、profile detail、claim 和 evidence response 字段。
- 前端新增对应 TypeScript 类型和只读展示页面。
- 不新增新的事实表，不改变 job/profile 删除、迁移、重建的 provenance 语义。

## 兼容性与迁移约束

- 新字段通过 lazy migration 补齐，不需要手动迁移脚本。
- Memory Health、Field Map、Evidence Matrix 都是只读 derived artifacts，不会直接覆盖 facts。
- 写回校验会更保守地丢弃无证据或 audit removed 的 claims；这属于预期行为变化，目的是避免长期记忆污染。
- 本次没有实现 `research_tasks`、`user_marks` 或 active reading curriculum，避免扩大事实源和删除/迁移复杂度。
