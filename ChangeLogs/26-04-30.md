# 2026-04-30 Memory 工作台与动态综述交互修复

## 背景与目标

本次更新针对 `specs/memory-260430.md` 中确认的两个前端体验问题：Memory Workspace 信息密度高但入口解释不足，Living Survey 来源追溯暴露内部技术 key 且缺少阅读引导。同时清理了 `RunPage.tsx` 中既有的 React hooks lint warning。

## 影响范围

- 规划文档：`specs/memory-260430.md`
- Memory Workspace：`web/src/pages/MemoryWorkspacePage.tsx`
- Living Survey：`web/src/pages/LivingSurveyPage.tsx`
- Run 页：`web/src/pages/RunPage.tsx`

## 核心改动

- 将 Memory Workspace 的一级入口按用途分为 `先读懂领域`、`查证据与问题`、`高级审计`，保留所有原有视图入口。
- 将 `当前冲突与待确认默认解` 改为全宽摘要卡，只显示待解决数量和含建议更新数量，并保留 `前往冲突队列` 入口。
- 将 `高价值 Claims` 改为独立全宽面板，保留点击展开证据预览和原始 Claim 跳转。
- 为 `实体分组` 增加解释，说明实体是 Memory 图谱稳定概念节点，标签后的数字代表关联 claims 数。
- 为 `领域认知摘要` 增加缺失类型说明，明确它展示的是高层 synthesis，不会把普通 claims 自动伪造成共识。
- 为动态综述增加阅读路径、中文化统计卡片、章节用途说明和更清晰的目录信息。
- 修复动态综述来源追溯显示 `Theme: theme:retrogfn` 的问题；内部 key 会格式化为可读标签，例如 `RetroGFN`，`coverage:theme:retrogfn` 会显示为 `覆盖空白 · RetroGFN`。
- 动态综述来源追溯在主题和空白章节优先显示 block 标题，技术 key 只作为兜底。
- 将页面标题从 `AI4Chem · Living Survey（动态综述）` 简化为 `AI4Chem · 动态综述`，避免标题位置中英混用。
- 修复 `RunPage.tsx` 的 `react-hooks/exhaustive-deps` warning：稳定化初始 tracked jobs 和 selected job 的计算，并让 effect 依赖真实数组。

## 数据结构 / API / 页面变化
