# 26-04-25 Manual Upload Topic Audit Skip + Configurable Similarity Thresholds

## Background & Goal

1. Manual PDF uploads with an explicitly selected profile were still subject to the post-download topic-fit audit, causing legitimate papers to be rejected as "off-topic" even though the user intentionally chose both the paper and the profile.
2. The topic-fit gate threshold (0.72) and post-download audit threshold (0.55) were hardcoded, requiring source code edits to tune. Users needed a frontend entry point to adjust these per-run or as browser defaults.

## Impact Scope

### Backend
- `server/job_manager.py` ŌĆö pipeline now passes `job_mode` and `selection` through `parsed_paper` to interpreter
- `modules/paper_interpreter/agent.py` ŌĆö skips post-download topic audit when `job_mode=manual` + `profile_mode=explicit` + valid `profile_id`; reads `post_download_topic_fit_threshold` from `parsed_paper["selection"]`
- `modules/paper_selector/agent.py` ŌĆö reads `topic_fit_gate_threshold` from `selection` config instead of hardcoded constant; includes threshold in selector diagnostics
- `server/schemas.py` ŌĆö `SelectionConfig` gains `topic_fit_gate_threshold` (default 0.72) and `post_download_topic_fit_threshold` (default 0.55)
- `utils/config.py` ŌĆö `DEFAULT_CONFIG["selection"]` gains both new fields with backward-compatible defaults

### Frontend
- `web/src/api/client.ts` ŌĆö `JobConfigOverride.selection` and `JobSelectionConfig` include new threshold fields
- `web/src/lib/jobConfig.ts` ŌĆö `PersistedRunPageState` and `NormalizedRunConfig` include `topicFitGateThreshold` / `postDownloadTopicFitThreshold`
- `web/src/pages/RunPage.tsx` ŌĆö new "Advanced Similarity Controls" card with two number inputs; values sent in `config_override.selection` for both auto and manual modes; persisted in session state
- `web/src/pages/SettingsPage.tsx` ŌĆö new default fields for both thresholds in "Default Run Config" card
- `web/src/pages/ReportsPage.tsx` ŌĆö `RetryDraftState` includes threshold fields; "Run Again" propagates saved thresholds

### Tests
- `tests/test_interpreter_smoke.py` ŌĆö existing smoke test now passes `job_mode` and `selection`; new test `test_manual_explicit_profile_skips_post_download_topic_audit` verifies skip behavior and threshold propagation

## Core Changes

### 1. Manual + Explicit Profile ŌåÆ Skip Post-Download Topic Audit

When the interpreter detects `job_mode=manual`, `profile_mode=explicit`, and a valid `profile_id`, it:
- Sets `topic_audit.skipped = True` with `reason = "manual_upload_with_explicit_profile"`
- Records `topic_audit.threshold` for transparency
- Sets working memory metric `selected_paper_topic_fit_skipped = 1.0`
- Persists the audit artifact (with skip marker) to `selector_diagnostics.json`
- Does **not** raise `RuntimeError` regardless of score

In all other cases (auto mode, manual+auto-assign), the post-download audit runs as before, using the configurable threshold.

### 2. Configurable Thresholds

| Threshold | Default | Where used | Config key |
|-----------|---------|------------|------------|
| Topic Fit Gate | 0.72 | Selector: filters candidates before LLM Top-1 | `selection.topic_fit_gate_threshold` |
| Post-download Audit | 0.55 | Interpreter: rejects paper after download if score below | `selection.post_download_topic_fit_threshold` |

Both are:
- Available in `config.yaml` under `selection`
- Overridable per-run via `config_override.selection`
- Exposed as browser-local defaults in Settings page
- Propagated in Run page session persistence and Reports "Run Again" flow
- Included in selector diagnostics (`topic_fit_gate_threshold`) and post-download audit (`threshold`) for post-hoc inspection

## Validation

- Frontend: `npm run build` passes
- Backend: smoke test + new manual-skip test added (environment lacked `boto3`/`httpx` for full pytest run)
- Backward compatibility: old configs without the new fields fall back to the original hardcoded defaults (0.72 / 0.55)

- No API breaking changes ŌĆö new fields have defaults
