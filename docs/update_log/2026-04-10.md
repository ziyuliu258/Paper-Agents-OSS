# 2026-04-10

## 背景

本次更新集中补齐 4 个核心能力：

1. profile 级级联删除
2. selector 的 precision-first 纠偏
3. 报告独立审查层
4. 未指定 profile 时的自动匹配 / 自动创建

同时同步更新 Web 工作台文档、仓库导航文档和现行规格，避免代码与说明脱节。

同日追加一轮 source-document 修复，解决“下载到的是 HTML 跳转页，却被误当成 PDF 进入 processor”这一类失败。

同日晚些时候，又补上一轮 profile memory 运营能力：支持把某个 profile 下选中的论文批量迁移到另一个 profile，并在迁移后重建 source / target 两边长期记忆。

## 主要实现点

### 1. Profile 删除闭环

- 新增 `DELETE /api/profiles/{profile_id}`
- `default` profile 禁止删除
- 若目标 profile 下存在活跃 job，接口返回 `409`，不自动强停
- 删除时会复用现有 job purge 逻辑，级联清理：
  - job record
  - paper record
  - `results/jobs/{job_id}`
  - `data/fetch/jobs/{job_id}`
  - `data/cache/jobs/{job_id}`
  - 该 job 写回的 memory bundle
- 最后再清理残余 profile-scoped memory 与 profile 自身
- 前端新增 destructive dialog，必须输入 `DELETE <profile_name>` 才能确认

### 2. 搜索改成 precision-first

- rerank 从“取最大值”改成 embedding + lexical blended score
- profile memory 不再直接混进 lexical overlap，只保留为 tie-break / 偏好提示
- 若 topic 缺少可检索英文关键词，后端会自动补英文 search hints；优先走启发式词表，必要时再调用模型生成
- selector 新增 `topic_fit_judge`
- 只有通过 fit 门槛的候选才会进入 `select_top_paper`
- 若全部候选被 topic-fit 拦下，job 直接失败，不再强行挑一篇
- `paper_notes` 完成后再做一次 selected paper topic audit，避免下载后才发现文章跑题
- `selector_diagnostics.json` 扩展记录：
  - raw candidates
  - reranked candidates
  - fit judgments
  - rejected candidates
  - failure reason
  - selected paper topic audit

### 3. Report audit 独立落层

- 新增 `modules/paper_interpreter/report_auditor.py`
- 审查插入在 `run_all_tasks()` 之后、memory writeback 与 assemble 之前
- 当前实现两类审查：
  - groundedness audit：结构化 claim 是否存在有效 evidence / page anchor
  - consistency audit：T1/T5/T6/T7 的数字说法是否与结构化 section 对得上
- 高风险 groundedness 问题会先做一次保守修复，再复审一次
- 审查结果写入 `results/jobs/{job_id}/report_audit.json`
- ReportViewer / Reports 现在可以感知 audit artifact 与 warning 状态

### 4. Auto profile routing

- job 新增 `profile_mode`: `explicit | auto`
- Run 页未选择 profile 时改为 `Auto assign`
- auto 模式先允许 selector 用 topics 做 soft-route，借用现有 profile 的检索记忆
- `paper_notes` 完成后再做最终归属：
  - 命中已有 profile：`Auto matched`
  - 无稳定命中：自动新建 profile，名称采用“主领域 + 主任务”
- `default` 不会被自动路由选中，只有显式选择才会使用
- 历史页、报告页、作业状态补充：
  - `profile_mode`
  - `profile_assignment_status`
  - `profile_assignment_note`
  - `has_report_audit`

### 5. HTML source 兜底链路

- selector 下载阶段新增 `utils/source_documents.py`
- 若 landing page / redirect page 最终能跳到真实 PDF，会继续保存为 PDF
- 若最终只能拿到 HTML，则保存 `source_path=.html`、`source_type=html`
- `papers` 表新增 `source_path`、`source_type`
- `PaperProcessorAgent` 新增 HTML 分支：
  - 不再做 figure identification
  - 改为抽取 `title / abstract / sections / plain_text`
- `build_paper_notes` 新增 HTML 分支，使用 `html_bundle` 做 text-only 结构化笔记生成
- `report_auditor` 新增 HTML 兼容，不再把 section 数量误当成 PDF 页数做 page bound 校验
- Papers 页、Profile Detail 里的论文入口会根据 source 类型显示 `Open PDF` 或 `Open Source`
- Reports 页对 failed auto job 新增 `Run Again` 按钮：带回原配置，但不会复用旧 fetch/source 文件

### 6. 双语 profile fingerprint 与跨语言路由

- profile assignment 不再主要依赖英文 token
- 新增双语 fingerprint：会同时吸收 profile 的 name / description / brief / 近期 activity 里的中英文片段
- 新增 embedding 匹配层：对论文上下文和 profile fingerprint 做向量相似度比较
- 最终分数由 embedding 相似度、规则语义分数和成熟 profile 偏好共同组成
- 当 topic 早已 soft-route 到某个现有成熟 profile 时，最终归档会优先复用该 profile，避免再创建一个语义相同的英文 profile 分身

### 7. Profile 间批量迁移 paper bundle

- 新增 `POST /api/profiles/{profile_id}/move-papers`
- Profile Detail 支持批量勾选当前 profile 下的文章 / job bundle，并迁移到另一个 profile
- 迁移语义不是只改 job 表头：
  - 会同步迁移 `memory_writebacks`
  - 迁移该 writeback 产出的 claims / synthesis / graph edges
  - 对被这些 claims 引用的 entity 做 move / clone / relink / merge
- 若目标 profile 已存在同名 entity 或相同 paper-edge，会尽量复用或合并，减少重复对象
- 迁移完成后会分别重建 source / target 两个 profile 的 cognition 与长期记忆快照
- 对应 job 会同步改成显式归属目标 profile，并写入手动迁移说明，避免报告页 / 历史页仍显示旧 profile

## 已知风险

- 当前 report audit 仍是保守规则审查，不是“绝对事实验证器”
- auto profile routing 目前是轻量 fingerprint 匹配，适合先解决“未指定 profile 时的自动归档”问题；后续可继续换成 embedding / structured rubric / learned router
- selector 的 topic-fit gate 先用规则和语义信号拦截明显跑题论文，后续仍可继续接入更强的 task-schema 审核器
- HTML 提取目前偏向正文与摘要恢复，不等价于完整 PDF 版式理解；如果站点正文裁剪很重，图表与精确页码锚点仍会弱于 PDF
- profile 批量迁移当前是“按 job bundle 精确迁移 + 目标侧必要合并”的保守实现；如果未来要支持更复杂的人工编辑 lineage，可继续把 revision / review 迁移规则做得更细

## 后续建议

- 为 report audit 增加更细的证据覆盖率指标与人工复核入口
- 将 auto profile routing 的阈值外提到配置，方便不同领域调优
- 给 selector diagnostics 做更直观的前端候选淘汰链路展示
- 给 profile 批量迁移增加更细的 dry-run 预览，比如预估会移动多少 claims / entities / graph edges
