# 2026-04-02 文档与现行规格同步

## 1. 背景

最近一轮实现已经把项目从“有 Web 工作台的论文解读器”进一步推进成了一个更完整的工作流系统：

- selector 新增 PDF URL enrichment 与更明确的 selectable-candidate 语义
- report 层新增 `classic / pmrc` 结构模式与 file-backed report variants
- job lifecycle 新增 `force-stop + purge`，并细化了 `retry` 与 `rerun / regenerate` 的语义
- Memory V2 已经支持按 job / paper 删除 provenance bundle

旧文档虽然已经大体描述了 Web-only 架构，但没有完整覆盖这些新行为，因此需要统一同步。

## 3. 这次明确写进文档的现行实现

### 3.1 selector 当前真实语义

- 支持 **venue-first** 与 **general search**
- 先做去重与 rerank，再通过 DOI landing page / HTML probe 补全 `pdf_url`
- 最终 selector 只在 **可下载 PDF 的候选** 中精排
- 支持 profile selection memory 影响 auto 选论文

### 3.2 interpreter 当前真实语义

- 先统一生成 `paper_notes`
- 使用 retrieval bundles 注入长期记忆
- 通过 WorkingMemory 保存运行内状态
- 采用 Group A / Group B 的 T1-T7 调度
- 在写回长期记忆前先做 distillation 与 promotion funnel

### 3.3 report 当前真实语义

- 生成 `report.en.md` 与 `report.md`
- 支持 `classic / pmrc` 两种结构模式
- 支持 working memory / distilled summary / selector diagnostics 三类工作台面板
- 支持 grounded report refinement，并将结果写成 file-backed variants

### 3.4 job lifecycle 当前真实语义

- `retry`：failed job 原地重试，同一个 `job_id`
- `rerun / regenerate`：新建 replacement job，成功后清理旧 job
- `force-stop`：强停并 purge 当前 job 的 artifacts、DB 记录和 memory bundle

## 4. 文档策略调整

这次同步后，文档分工更明确：

- `README.md`：面向外部用户的总体介绍与快速上手
- `AGENTS.md` / `CLAUDE.md`：面向 agent / coder 的仓库导航与不变量
- `specs/Web工作台与记忆系统-现行规格.md`：当前实现的正式规格
- `specs/上下文与记忆.md`：当前上下文与记忆分层说明
- 历史规划类文档继续保留，但在顶部明确标注其“历史背景”属性

这次改动主要是 **文档与规格同步**，不是新的后端功能提交；目标是让当前仓库里已经存在的真实实现能够被 README、agent 指南和 spec 正确描述。
